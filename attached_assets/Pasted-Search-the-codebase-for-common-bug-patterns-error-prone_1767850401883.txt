The Problem
The decryptLegacy function had an empty catch block that silently swallowed all decryption errors:

} catch {
  return null;
}

This made debugging migration issues very difficult because:

Errors were completely hidden with no logging
Couldn't distinguish between legitimate decryption failures and unexpected errors
Made troubleshooting encrypted data problems nearly impossible
The Solution
Added proper error logging while maintaining the same behavior:

} catch (error) {
  console.error('Legacy decryption failed:', {
    error: error.message,
    encryptedLength: encrypted?.length,
    encryptedPrefix: encrypted?.substring(0, 10) + '...'
  });
  return null;
}

Now the function logs:

The specific error message
Length of the encrypted data (helps identify corruption)
A prefix of the encrypted data (for debugging without exposing full content)
